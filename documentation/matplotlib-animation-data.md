# Animations using Matplotlib[#](#animations-using-matplotlib "Link to this heading")

Based on its plotting functionality, Matplotlib also provides an interface to generate animations using the [animation](../../../api/animation%5Fapi.html#module-matplotlib.animation "matplotlib.animation") module. An animation is a sequence of frames where each frame corresponds to a plot on a[Figure](../../../api/%5Fas%5Fgen/matplotlib.figure.Figure.html#matplotlib.figure.Figure "matplotlib.figure.Figure"). This tutorial covers a general guideline on how to create such animations and the different options available. More information is available in the API description: [animation](../../../api/animation%5Fapi.html#module-matplotlib.animation "matplotlib.animation")

import matplotlib.pyplot as plt
import numpy as np

import matplotlib.animation as animation

## Animation classes[#](#animation-classes "Link to this heading")

The animation process in Matplotlib can be thought of in 2 different ways:

* [FuncAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.FuncAnimation.html#matplotlib.animation.FuncAnimation "matplotlib.animation.FuncAnimation"): Generate data for first frame and then modify this data for each frame to create an animated plot.
* [ArtistAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.ArtistAnimation.html#matplotlib.animation.ArtistAnimation "matplotlib.animation.ArtistAnimation"): Generate a list (iterable) of artists that will draw in each frame in the animation.

[FuncAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.FuncAnimation.html#matplotlib.animation.FuncAnimation "matplotlib.animation.FuncAnimation") is more efficient in terms of speed and memory as it draws an artist once and then modifies it. On the other hand [ArtistAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.ArtistAnimation.html#matplotlib.animation.ArtistAnimation "matplotlib.animation.ArtistAnimation") is flexible as it allows any iterable of artists to be animated in a sequence.

### `FuncAnimation`[#](#funcanimation "Link to this heading")

The [FuncAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.FuncAnimation.html#matplotlib.animation.FuncAnimation "matplotlib.animation.FuncAnimation") class allows us to create an animation by passing a function that iteratively modifies the data of a plot. This is achieved by using the _setter_ methods on various[Artist](../../../api/artist%5Fapi.html#matplotlib.artist.Artist "matplotlib.artist.Artist") (examples: [Line2D](../../../api/%5Fas%5Fgen/matplotlib.lines.Line2D.html#matplotlib.lines.Line2D "matplotlib.lines.Line2D"),[PathCollection](../../../api/collections%5Fapi.html#matplotlib.collections.PathCollection "matplotlib.collections.PathCollection"), etc.). A usual[FuncAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.FuncAnimation.html#matplotlib.animation.FuncAnimation "matplotlib.animation.FuncAnimation") object takes a[Figure](../../../api/%5Fas%5Fgen/matplotlib.figure.Figure.html#matplotlib.figure.Figure "matplotlib.figure.Figure") that we want to animate and a function_func_ that modifies the data plotted on the figure. It uses the _frames_parameter to determine the length of the animation. The _interval_ parameter is used to determine time in milliseconds between drawing of two frames. Animating using [FuncAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.FuncAnimation.html#matplotlib.animation.FuncAnimation "matplotlib.animation.FuncAnimation") typically requires these steps:

1. Plot the initial figure as you would in a static plot. Save all the created artists, which are returned by the plot functions, in variables so that you can access and modify them later in the animation function.
2. Create an animation function that updates the artists for a given frame. Typically, this calls `set_*` methods of the artists.
3. Create a [FuncAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.FuncAnimation.html#matplotlib.animation.FuncAnimation "matplotlib.animation.FuncAnimation"), passing the [Figure](../../../api/%5Fas%5Fgen/matplotlib.figure.Figure.html#matplotlib.figure.Figure "matplotlib.figure.Figure") and the animation function.
4. Save or show the animation using one of the following methods:  
   * [pyplot.show](../../../api/%5Fas%5Fgen/matplotlib.pyplot.show.html#matplotlib.pyplot.show "matplotlib.pyplot.show") to show the animation in a window  
   * [Animation.to\_html5\_video](../../../api/%5Fas%5Fgen/matplotlib.animation.Animation.html#matplotlib.animation.Animation.to%5Fhtml5%5Fvideo "matplotlib.animation.Animation.to_html5_video") to create a HTML `<video>` tag  
   * [Animation.to\_jshtml](../../../api/%5Fas%5Fgen/matplotlib.animation.Animation.html#matplotlib.animation.Animation.to%5Fjshtml "matplotlib.animation.Animation.to_jshtml") to create HTML code with interactive JavaScript animation controls  
   * [Animation.save](../../../api/%5Fas%5Fgen/matplotlib.animation.Animation.html#matplotlib.animation.Animation.save "matplotlib.animation.Animation.save") to save the animation to a file

### `ArtistAnimation`[#](#artistanimation "Link to this heading")

[ArtistAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.ArtistAnimation.html#matplotlib.animation.ArtistAnimation "matplotlib.animation.ArtistAnimation") can be used to generate animations if there is data stored on various different artists. This list of artists is then converted frame by frame into an animation. For example, when we use [Axes.barh](../../../api/%5Fas%5Fgen/matplotlib.axes.Axes.barh.html#matplotlib.axes.Axes.barh "matplotlib.axes.Axes.barh") to plot a bar-chart, it creates a number of artists for each of the bar and error bars. To update the plot, one would need to update each of the bars from the container individually and redraw them. Instead, [animation.ArtistAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.ArtistAnimation.html#matplotlib.animation.ArtistAnimation "matplotlib.animation.ArtistAnimation") can be used to plot each frame individually and then stitched together to form an animation. A barchart race is a simple example for this.

## Animation writers[#](#animation-writers "Link to this heading")

Animation objects can be saved to disk using various multimedia writers (ex: Pillow, _ffpmeg_, _imagemagick_). Not all video formats are supported by all writers. There are 4 major types of writers:

* [PillowWriter](../../../api/%5Fas%5Fgen/matplotlib.animation.PillowWriter.html#matplotlib.animation.PillowWriter "matplotlib.animation.PillowWriter") \- Uses the Pillow library to create the animation.
* [HTMLWriter](../../../api/%5Fas%5Fgen/matplotlib.animation.HTMLWriter.html#matplotlib.animation.HTMLWriter "matplotlib.animation.HTMLWriter") \- Used to create JavaScript-based animations.
* Pipe-based writers - [FFMpegWriter](../../../api/%5Fas%5Fgen/matplotlib.animation.FFMpegWriter.html#matplotlib.animation.FFMpegWriter "matplotlib.animation.FFMpegWriter") and[ImageMagickWriter](../../../api/%5Fas%5Fgen/matplotlib.animation.ImageMagickWriter.html#matplotlib.animation.ImageMagickWriter "matplotlib.animation.ImageMagickWriter") are pipe based writers. These writers pipe each frame to the utility (_ffmpeg_ / _imagemagick_) which then stitches all of them together to create the animation.
* File-based writers - [FFMpegFileWriter](../../../api/%5Fas%5Fgen/matplotlib.animation.FFMpegFileWriter.html#matplotlib.animation.FFMpegFileWriter "matplotlib.animation.FFMpegFileWriter") and[ImageMagickFileWriter](../../../api/%5Fas%5Fgen/matplotlib.animation.ImageMagickFileWriter.html#matplotlib.animation.ImageMagickFileWriter "matplotlib.animation.ImageMagickFileWriter") are examples of file-based writers. These writers are slower than their pipe-based alternatives but are more useful for debugging as they save each frame in a file before stitching them together into an animation.

To save animations using any of the writers, we can use the[animation.Animation.save](../../../api/%5Fas%5Fgen/matplotlib.animation.Animation.html#matplotlib.animation.Animation.save "matplotlib.animation.Animation.save") method. It takes the _filename_ that we want to save the animation as and the _writer_, which is either a string or a writer object. It also takes an _fps_ argument. This argument is different than the_interval_ argument that [FuncAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.FuncAnimation.html#matplotlib.animation.FuncAnimation "matplotlib.animation.FuncAnimation") or[ArtistAnimation](../../../api/%5Fas%5Fgen/matplotlib.animation.ArtistAnimation.html#matplotlib.animation.ArtistAnimation "matplotlib.animation.ArtistAnimation") uses. _fps_ determines the frame rate that the**saved** animation uses, whereas _interval_ determines the frame rate that the **displayed** animation uses.
